「로봇활용 유도탄(천궁Ⅱ) 도장자동화시스템 구축」  프로젝트 UI 디자인
관리자 페이지 및 예지보전시스템 UI 디자인을 담당했습니다. 3920×2205px 대형 커스텀 해상도 환경에 맞춰 화면 구조를 설계하고, 산업 설비 데이터를 효율적으로 시각화하는 UI를 구현했습니다.

GU VINA MES 프로젝트의 주요 화면 퍼블리싱을 담당하였으며, 4K UHD부터 PC 해상도까지 대응하는 반응형 UI를 구현했습니다. 작업지시 화면을 포함한 전체 화면 구조 설계 및 공통 스타일 정리를 통해 일관된 퍼블리싱 구조를 구축했습니다.

  • HTML / CSS
  • - 접근성과 유지보수를 고려한 Semantic Markup 설계
  • - PC·Tablet·Mobile 환경에 최적화된 반응형 레이아웃 구현
  • - 사용자 흐름을 자연스럽게 만드는 CSS Animation 및 인터랙션 적용
  • JavaScript
  • - jQuery 및 Vanilla JavaScript를 활용한 UI 인터랙션 구현
  • - DOM 제어 및 조건 분기를 통한 동적 기능 구현
  • - GSAP / ScrollTrigger 기반 스크롤 애니메이션 적용
  • UI Library
  • - Swiper/Slick : 반응형 슬라이드, 커스텀 네비게이션 구현
  • - AOS : 스크롤 위치에 따른 콘텐츠 등장 애니메이션 적용
  • - Chart.js : 관리자 페이지 내 데이터 시각화 차트 적용
  • Collaboration
  • - Git 기반 협업 환경에서 퍼블리싱 작업 진행
  • - 기획·디자인·개발 간 커뮤니케이션 경험
  • - 공통 컴포넌트 관리 경험

# 웹애니메이션 # 프론트엔드
문제 해결 및 개선 경험
ScrollTrigger 초기 렌더링 및 Refresh 타이밍 최적화
ScrollTrigger 적용 시 새로고침 직후 요소가 순간적으로 튀는 초기 렌더링 이슈를 경험했습니다.
DOM 렌더링 타이밍과 ScrollTrigger의 refresh 시점 차이에서 원인을 파악하고,
초기 상태 설정 및 옵션 조정을 통해 레이아웃 계산을 안정화했습니다. 이를 통해 스크롤 인터랙션이 자연스럽게 이어지도록 개선했습니다.
모바일 환경에 맞춘 인터랙션 구조 개선
PC hover 기반 인터랙션이 모바일 환경에서 동작하지 않는 문제를 인지하고, 디바이스 환경에 따라 hover와 click 이벤트를 분리 적용했습니다.
이를 통해 모든 환경에서 일관되고 자연스러운 사용자 경험을 구현했습니다.
스크롤 이벤트 최소화 및 퍼포먼스 개선
과도한 scroll 이벤트 사용으로 인한 성능 저하를 개선하기 위해 불필요한 이벤트를 제거하고 GSAP ScrollTrigger 기반 구조로 전환했습니다.
그 결과, 렌더링 부담을 줄이고 부드러운 스크롤 인터랙션을 구현했습니다.
팝업 내 Swiper 초기화 문제
팝업 내부에 Swiper 적용 시 초기 레이아웃 계산 오류로 화면이 깨지는 문제를 경험했습니다.
팝업 오픈 이후 슬라이더를 재초기화하도록 로직을 수정하여 안정적인 레이아웃을 유지하도록 개선했습니다.
클릭 영역 충돌 문제
카드 전체를 감싸는 링크 구조로 인해 내부 버튼 클릭이 차단되는 이슈를 확인했습니다.
z-index 구조와 pointer-events 속성을 재설계하여 사용자 클릭 흐름이 자연스럽게 동작하도록 개선했습니다.
유지보수를 고려한 리팩토링
중복된 CSS와 복잡한 구조로 인해 확장성이 낮았던 부분을 정리하고,
공통 스타일 분리 및 구조 재정비를 통해 유지보수성과 재사용성을 향상시켰습니다.
